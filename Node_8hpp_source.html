117  // we have a private pointer so different types can share
118  // in manner not exacctly congruent with type hierarchy
119  // in particular, `Inst` and `Load` want `User` lists
120  // while `Stow`s do not.
121  // `Addr` is the common load/store subtype
122  // So in some sense, we want both `Load` and `Store` to inherit from `Addr`,
123  // but only load to inherit 'hasUsers' and only store to inherit the operand.
124  // `Inst` would also inherit 'hasUsers', but would want a different operands
125  // type.
126  // Addr has a FAM, so multiple inheritance isn't an option for `Load`/`Stow`,
127  // and we want a common base that we can query to avoid monomorphization.
