Detailed Description

template<class T, class S, size_t N, class A, std::unsigned_integral U>
struct LinAlg::ManagedArray< T, S, N, A, U >

Stores memory, then pointer. Thus struct's alignment determines initial alignment of the stack memory. Information related to size is then grouped next to the pointer.

The Intel compiler + OpenMP appears to memcpy data around, or at least build ManagedArrays bypassing the constructors listed here. This caused invalid frees, as the pointer still pointed to the old stack memory.
