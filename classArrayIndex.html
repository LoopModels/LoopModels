constexpr auto ArrayIndex::indexMatrix ( ) -> MutDensePtrMatrix<int64_t>
inlineconstexpr

indexMatrix() -> getNumLoops() x arrayDim() loops are in [outermost <-> innermost] order Maps loop indVars to array indices Letting i be the indVars and d the indices: indexMatrix()' * i == d e.g. indVars = [i, j] and indexMatrix = [ 1 1; 0 1] corresponds to A[i, i + j] Note that [i, j] refers to loops in innermost -> outermost order, i.e. for (i : I) for (j : J) A[i, i + j]
