Detailed Description

POD. Gives counts for the different kinds of costs. Fields: int16_t trip_count- we're unlikely to change decisions for >32k negative indicates compile-time known size. uint16_t memory number of mem sets. bool exit loop exit/entry. uint31_t compute number of compute sets. These give us info for iterating over the costs associated with a loop. for (i : I){ for (j : J){ for (k : K){ // leaf ... } for (k : K){ // leaf ... } } for (j : J){ // leaf ... } } For leaves, we compute latency as well as register cost. Note that we compute all costs at the header for a given depth, thus we only need headers and num-pops.
